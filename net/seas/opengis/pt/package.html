Positioning implementation. Class {@link net.seas.opengis.pt.CoordinatePoint} represents a point in a multi-dimensional space. This space may have an arbitrary number of dimensions. For a two-dimensional space, CoordinatePoint is conceptually equivalent to {@link java.awt.geom.Point2D}.

Class {@link net.seas.opengis.pt.Envelope} represents a box in a multi-dimensional space. For a two-dimensional space, Envelope is conceptually equivalent to {@link java.awt.geom.Rectangle2D}.

 

 

 

Differences with the OpenGIS specification


 
  • Omitted the PT_ prefix for all classes and interfaces.
     
  • Omitted the {@link org.opengis.pt.PT_Matrix} class in order to avoid duplication with standard Java classes. Replacement is not yet choosen. It may be Jama.Matrix.
     
  • Added {@link net.seas.opengis.pt.ConvexHull} class. This class is used as argument type for {@link net.seas.opengis.ct.MathTransform#getDomainFlags} instead of a generic double[].
     
  • Added {@link net.seas.opengis.pt.Angle}, {@link net.seas.opengis.pt.Latitude} and {@link net.seas.opengis.pt.Longitude} classes for parsing and formatting with {@link net.seas.text.AngleFormat}.
     
  • Provide various methods and constructors to ease interoperability with the Java2D geometric package.

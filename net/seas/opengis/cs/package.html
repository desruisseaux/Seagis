Coordinate Systems implementation.

 

 

 

Differences with the OpenGIS specification

  • Omitted the CS_ prefix for all classes and interfaces.
  • Omitted {@link org.opengis.cs.CS_Unit} class and subclasses in order to avoid duplication with standard Java classes. Use {@link javax.units.Unit} instead.
  • Omitted {@link org.opengis.cs.CS_ProjectionParameter} in order to avoid duplication with {@link org.opengis.ct.CT_Parameter}. Use {@link net.seas.opengis.ct.Parameter} instead.
  • Made all enumeration classes immutable. Enumeration constants are instances of the actual enumeration class instead of integers. Names are uppercase, with no CS_AO prefix.
  • {@link net.seas.opengis.cs.Ellipsoid#getInverseFlattening} returns {@link java.lang.Double#POSITIVE_INFINITY} instead of 0 for perfect spheres. Infinity is the mathematically correct value.
  • Added methods:
    • {@link net.seas.opengis.cs.Ellipsoid#getEccentricity}
    • {@link net.seas.opengis.cs.Ellipsoid#orthodromicDistance}

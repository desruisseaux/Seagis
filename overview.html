Provides implementation for general positioning, coordinate systems, and coordinate transformations. SEAGIS is an open source implementation of a subset of OpenGIS® Coordinate Transformation Services Specification and OpenGIS® Grid Coverages Specification. Being OpenGIS-compliant should allow developers to switch between this free implementation and any commercial implementation with no code change (providing that commercial implementations are OpenGIS compliant).

 

Packages organization

A SEAGIS's goal is to ensure interoperability with standard Java packages like Java2D and Java Advanced Imaging. However, it is not always possible to leverage Java classes and be OpenGIS-compliant in the same time. SEAGIS workaround the problem by providing two sets of packages:

  • The org.opengis package is the standard OpenGIS specification. Classes in this package are mostly interfaces, with no implementation. Interfaces defined there are well suited for RMI use, which implies that every method is declared to throws {@link java.rmi.RemoteException}. Developers looking for strict OpenGIS compliance and/or RMI use should look at this org.opengis package.
     
  • The net.seas.opengis package is an org.opengis mirror tuned for Java and local use. It contains concrete classes (not interfaces). There is generally a 1 to 1 correspondence between OpenGIS interfaces and SEAGIS classes, so developers familiar with one package should be familiar with the other. SEAGIS methods do not thrown RemoteException, which make them easier for local use but is not suitable for RMI use. Developers looking for Java2D interoperability and/or local use may look at this net.seas.opengis package.
     

Factory classes allow the creation of OpenGIS objects (RMI enabled) backed by SEAGIS classes. Internally, interoperability between OpenGIS interfaces and SEAGIS classes is assured through a set of RMI-aware adapters. Classes not related to any OpenGIS API appear in separated packages (e.g. net.seas.map) and sub-packages.

 

Additional reading: A list of differences between SEAGIS's API and OpenGIS specification.

 

Note about fr.ird packages

The fr.ird branch contains experimental classes needed for some work at IRD (Institut de Recherche pour le Développement, France). It can be seen as an example of use of net.seas packages. Note that the net.seas branch has no dependencies to fr.ird. Consequently, the whole fr directory and sub-directories can be deleted (if you want) without any impact on the OpenGIS implementation in net.seas.
